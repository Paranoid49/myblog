# 项目哲学 9.0+ 优化执行清单

日期：2026-03-14

## 1. 文档目标

本计划文档用于把“项目哲学评分从 8.2 提升到 9.0+”的路线图拆解为可执行 Todo 清单。

优化目标遵循项目核心哲学：

- 轻量
- 克制
- 可扩展

原则上：

> 不通过增加功能和抽象来提分，而通过降低噪音、收紧边界、固化扩展契约与测试来提分。

---

## 2. 评分提升目标

当前评估：

- 轻量：8.5/10
- 克制：8/10
- 可扩展：8/10
- 综合：8.2/10

目标状态：

- 轻量：8.8+
- 克制：9.0+
- 可扩展：8.8+
- 综合：9.0+

达成路径：

1. 先做“减噪收口”
2. 再做“边界固化”
3. 最后做“优雅收束”

---

## 3. 执行原则

### 3.1 轻量
- 优先删除噪音、统一入口、清理残留。
- 不引入新的基础设施、重型状态管理或平台型抽象。

### 3.2 克制
- 所有改动必须能说明其与当前边界直接相关。
- 不借优化之名做审美式重构。
- 历史材料归档，不让历史方案继续充当当前真相。

### 3.3 可扩展
- 保持已有扩展点最小可用：database provider、extension loader、hook bus、theme register。
- 补契约、补测试，不扩平台。

## 8. 哪些不要做

为了把分数稳定提升到 9.0+，以下方向当前不建议做：

- [ ] 不要把扩展机制升级为插件平台
- [ ] 不要为了“前端规范化”引入重型全局状态管理
- [ ] 不要为了数据库抽象继续下挖平台化适配层
- [ ] 不要借优化之名做大范围审美式重构
- [ ] 不要扩评论、多用户、多角色等超出当前边界的能力
- [ ] 不要继续增加大而全但缺乏维护的文档
