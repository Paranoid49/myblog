# 2026-03-09 阶段 3.5 最小产品化任务清单

## 目标

在不进入阶段 5（上线与开源准备）前，先把当前项目从“功能骨架”推进到“可展示、可使用、可继续打磨”的状态。

核心要求：
- 保持轻量，不引入重型 UI 框架
- 优先补齐阅读体验与写作体验
- 优先做用户可感知的质量提升
- 不做超出 V1 的复杂功能（评论、搜索、草稿协作、在线富文本等）

---

## 当前问题总结

### 产品层短板
- 前台首页、详情页、作者页视觉层级不足，像调试页面，不像博客产品
- 后台页面可用但界面粗糙，操作区、信息区、反馈区未分层
- 缺少统一布局与样式规范

### 内容层短板
- Markdown 目前只是在编辑/存储，前台详情页未真正渲染
- 后台编辑器缺少实时预览或预览切换
- 文章阅读体验不完整（标题、摘要、元信息、正文样式都偏弱）

### 开源层短板
- 当前截图和演示效果不足以支撑开源第一印象
- README 即使完善，项目视觉完成度仍会拖后腿

---

## 阶段 3.5：最小产品化打磨

### P3.5-1 Markdown 渲染闭环
- 目标：让 Markdown 从“可保存”升级为“可阅读”
- 范围：
  - 前台详情页支持 Markdown 渲染
  - 后台编辑页增加预览能力（切换预览或双栏预览，二选一）
  - 支持基础语法：标题、段落、列表、引用、代码块、图片、链接
- 建议实现：
  - 前端引入轻量 Markdown 渲染库
  - 避免引入重型富文本编辑器
- 验收：
  - `frontend/src/public/pages/PublicPostDetailPage.jsx` 不再使用 `pre` 直接输出正文
  - 后台编辑文章时可预览渲染结果
  - 上传图片插入后的 Markdown 可被正常渲染

### P3.5-2 前台公共布局重做
- 目标：让前台页面具备博客产品感
- 范围：
  - 抽出前台公共布局（头部 / 主体 / 页脚）
  - 首页增加更清晰的标题区、导航区、文章列表区
  - 主题切换按钮与导航入口统一风格
- 约束：
  - 不引入组件库
  - 继续使用现有 CSS variables 体系
- 验收：
  - `frontend/src/public/pages/PublicHomePage.jsx` 使用统一布局容器
  - 首页文章列表从简单 `ul/li` 升级为卡片或列表块结构
  - 暗黑/浅色主题下都具备可读性

### P3.5-3 文章详情页阅读体验优化
- 目标：让详情页达到“可正常阅读博客文章”的最低标准
- 范围：
  - 标题、摘要、正文区分层
  - 增加发布时间、分类/标签等元信息展示
  - 优化段落、标题、代码块、图片、引用样式
- 依赖：P3.5-1
- 验收：
  - 文章详情具备清晰的排版层级
  - Markdown 主要元素样式可读
  - 长文阅读时不会显得像原始文本转储

### P3.5-4 作者页产品化
- 目标：让作者页从“字段展示页”变成“个人主页最小版本”
- 范围：
  - 增加作者页头部信息区
  - 展示昵称、简介、邮箱（邮箱是否直接展示可再定）
  - 与首页、详情页保持统一视觉风格
- 验收：
  - `frontend/src/public/pages/PublicAuthorPage.jsx` 不再是简单段落堆叠
  - 访客可感知这是博客作者页，而非后台调试页

### P3.5-5 后台布局与导航整理
- 目标：让后台页面具备最低限度的管理系统体验
- 范围：
  - 抽出后台公共布局（头部、导航、主内容区）
  - 统一首页、文章管理、分类标签、作者资料页面结构
  - 统一按钮、表单、列表、反馈提示样式
- 约束：
  - 不引入后台模板框架
  - 优先通过现有 CSS 和少量组件拆分实现
- 验收：
  - `frontend/src/admin/pages/AdminHomePage.jsx`
  - `frontend/src/admin/pages/AdminPostsPage.jsx`
  - `frontend/src/admin/pages/AdminTaxonomyPage.jsx`
  - `frontend/src/admin/pages/AdminAuthorPage.jsx`
    都具有一致的顶部导航和内容区结构

### P3.5-6 后台文章编辑体验优化
- 目标：让后台写作页从“功能堆叠”变成“可持续使用”
- 范围：
  - 表单区、操作区、筛选区、列表区分块
  - Markdown 编辑区域提升可读性
  - 导入、导出、上传图片等操作集中到清晰的位置
  - 增加保存成功/失败的可见反馈
- 依赖：P3.5-1、P3.5-5
- 验收：
  - `frontend/src/admin/pages/AdminPostsPage.jsx` 结构清晰
  - 用户能快速分辨“编辑区”和“文章列表区”
  - 常用操作不再显得杂乱

### P3.5-7 前台列表信息补齐
- 目标：让首页具备博客内容入口价值
- 范围：
  - 首页文章项增加摘要
  - 增加发布时间、分类、标签的最小展示
  - 增加空状态、加载状态、错误状态优化
- 验收：
  - 首页不只是标题链接列表
  - 用户能快速判断文章内容和主题

### P3.5-8 主题样式体系补强
- 目标：让 light/dark 不只是颜色切换，而是完整视觉主题
- 范围：
  - 补充面板、边框、按钮、代码块、引用、链接等变量和样式
  - 保证前台与后台都走统一主题变量
- 依赖：P3.5-2、P3.5-5
- 验收：
  - `frontend/src/shared/theme/theme.css` 承载完整基础样式体系
  - 切换主题后前后台都保持一致观感

建议按影响力与依赖关系执行：

1. `P3.5-1 Markdown 渲染闭环`
2. `P3.5-3 文章详情页阅读体验优化`
3. `P3.5-2 前台公共布局重做`
4. `P3.5-7 前台列表信息补齐`
5. `P3.5-4 作者页产品化`
6. `P3.5-5 后台布局与导航整理`
7. `P3.5-6 后台文章编辑体验优化`
8. `P3.5-8 主题样式体系补强`

---

## 明确不做

以下内容暂不纳入阶段 3.5：
- 富文本编辑器
- 评论系统
- 全文搜索
- 文章统计分析
- 多用户协作
- 在线草稿自动保存
- 复杂动画与花哨视觉特效
- 重型 UI 组件库整体接入

原因：这些内容会显著增加复杂度，偏离当前“小而美、轻量化”的目标。

---

## 完成标准

满足以下条件后，再考虑进入阶段 5：
- 前台首页、详情页、作者页具备基本产品观感
- Markdown 已形成完整渲染闭环
- 后台写作页面具备最小可用体验
- light/dark 主题在前后台都表现稳定
- 可以截图、录屏、演示，而不会明显暴露“调试界面感”
