# 后续优化 TODO 清单

日期：2026-03-12

## 说明

本清单用于整理项目下一阶段的优化方向，遵循项目哲学：

- 轻量
- 克制
- 可扩展

原则上：

> 先把现有能力跑稳、跑顺、跑一致，再考虑新增功能。

---

## P1：优先立即做

### TODO 1：收口开发态前后端联调约定
**状态：已完成**
**关联文档**：`docs/dev-fullstack-contract.md`

**要做的事**
- [ ] 明确 `start_fullstack.py` 是推荐开发启动入口
- [ ] 明确前端 dev 模式 API 请求策略（相对路径 + Vite proxy，或固定约定）
- [ ] 明确后端端口、前端端口、静态资源访问方式
- [ ] 补一份简短开发约定文档（建议放 `docs/`）

**验收标准**
- 新人只按文档启动一次，就能完成前后端联调
- 不再反复出现“本地能开但行为不一致”的问题

---

### TODO 2：补关键浏览器冒烟测试
**状态：已完成**
**目标**：覆盖少量高价值用户路径。

**优先补的流程**
- [ ] 登录
- [ ] 登录后访问 `/admin/login` 自动回跳
- [ ] 登录后从首页点击“后台”进入 `/admin`
- [ ] 新建文章
- [ ] 发布文章
- [ ] 前台查看文章详情

**验收标准**
- 这些流程可一键跑
- 能快速发现“接口没问题但页面链路有问题”的 bug

---

### TODO 3：统一后台入口行为
**状态：已完成**
**目标**：所有进入后台的入口逻辑一致。

**要做的事**
- [ ] 检查 PublicLayout 中“后台”入口
- [ ] 检查首页、作者页、其他可能进入后台的链接
- [ ] 检查登录页本身的重定向逻辑
- [ ] 检查本地登录态与页面跳转的一致性

**验收标准**
- 未登录 → 后台入口去 `/admin/login`
- 已登录 → 后台入口去 `/admin`
- 已登录用户访问登录页会自动跳 `/admin`

### TODO 5：统一前端认证体验细节
**状态：已完成**
**目标**：减少“已登录但体验不一致”的问题。

**要做的事**
- [ ] 检查登录后 localStorage 写入是否统一
- [ ] 检查登出后 localStorage 清理是否统一
- [ ] 检查后台入口、登录页、AuthGuard 对同一登录态的判断是否一致
- [ ] 检查刷新页面后后台页行为是否符合预期

**验收标准**
- 登录、登出、后台入口、刷新后的行为一致
- 不再出现“看起来像登录了，但入口还像没登录”的问题

## 哪些不要做

为了保持项目哲学，下面这些现在不建议做：

- [ ] 不要引入 JWT 替代 session
- [ ] 不要把当前后端拆成大量细碎模块
- [ ] 不要为了“优雅”重构一大圈
- [ ] 不要把主题系统升级成重型插件系统
- [ ] 不要扩评论、多用户、多角色等超出当前边界的能力
