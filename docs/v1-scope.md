# V1 功能边界与架构原则

## 1. 项目定位

- 目标：个人独立博客系统
- 核心原则：小而美、轻量化、高性能、低维护
- 开源策略：核心保持简洁，扩展能力对社区开放

## 2. V1 必做范围（In Scope）

### 2.1 内容系统
- 文章创建/编辑/发布/转草稿
- Markdown 正文存储与渲染
- Markdown 单文件导入（`.md`）
- Markdown 单篇导出（`.md`）
- 基于 slug 的稳定文章链接

### 2.2 分类与标签
- 分类单选
- 标签多选
- 后台管理分类与标签
- 前台按分类/标签检索

### 2.3 附件（图片）
- 仅支持图片上传（jpg/png/webp/gif）
- 上传后返回 URL，可插入 Markdown
- 基础安全限制（类型、大小、重命名防冲突）

### 2.4 主题
- 内置 light/dark 两套主题
- 一键切换与本地持久化

### 2.5 作者资料
- 单管理员公开资料页
- 后台可编辑作者资料（头像/昵称/简介/链接）

## 3. V1 不做范围（Out of Scope）

- 评论系统默认不上线（后续可选扩展）
- 多用户、多角色权限模型
- 富文本编辑器与协作编辑
- 定时发布、批量导入导出
- 重型基础设施（Redis/ES/MQ/K8s）

## 4. 架构原则

### 4.1 前后端分离
- 前端：React（管理端优先迁移，前台随后迁移）
- 后端：FastAPI 仅提供 JSON API（`/api/v1/*`）
- 统一响应结构：`code / message / data`

### 4.2 轻量优先
- 默认单体部署
- 默认 SQLite
- 认证第一阶段继续使用 session/cookie（same-origin）

### 4.3 扩展优先
- 核心内核最小化
- 可替换数据库 provider
- 可替换主题 provider
- 关键业务事件提供 Hook 机制

## 5. 官方支持基线（V1）

- 数据库：SQLite、PostgreSQL
- 主题：light、dark
- 其余数据库/主题由社区扩展实现

## 6. 验收口径

满足以下条件即视为 V1 范围达标：
1. 可写：Markdown 文章创建、编辑、发布
2. 可看：首页/详情/分类/标签访问稳定
3. 可管：分类标签与作者资料可维护
4. 可插图：图片上传并在文章中可见
5. 可扩展：数据库与主题具备明确扩展入口
