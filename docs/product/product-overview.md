# 产品与范围总览

## 项目定位

myblog 是一个面向个人使用场景的独立博客系统，目标不是做大而全的平台，而是构建一个：

- 轻量
- 克制
- 可扩展

的最小生产闭环博客系统。

核心追求：

- 默认单体部署
- 默认轻量数据库方案
- 默认 session 认证
- 核心功能刚好够用
- 在不增加复杂度的前提下保留清晰扩展入口

---

## V1 范围

### 内容系统
- 文章创建 / 编辑 / 发布 / 转草稿
- Markdown 正文存储与渲染
- Markdown 单文件导入 / 单篇导出
- 基于 slug 的稳定文章链接

### 分类与标签
- 分类单选
- 标签多选
- 后台管理分类与标签
- 前台按分类 / 标签检索

### 图片能力
- 支持图片上传（jpg / png / webp / gif）
- 上传后返回 URL，可插入 Markdown
- 基础安全限制：类型、大小、重命名防冲突

### 主题能力
- 内置 `light` / `dark`
- 支持切换与本地持久化

### 作者资料
- 单管理员公开资料页
- 后台可编辑头像 / 昵称 / 简介 / 链接

---

## V1 不做范围

当前阶段明确不纳入：

- 评论系统
- 多用户 / 多角色权限模型
- 富文本编辑器与协作编辑
- 定时发布 / 批量导入导出
- Redis / ES / MQ / K8s 等重型基础设施

---

## 架构原则

### 前后端分离
- 前端：React SPA
- 后端：FastAPI 提供 JSON API，并托管 SPA 静态产物
- 统一响应结构：`code / message / data`

### 轻量优先
- 默认单体部署
- 默认 SQLite
- 认证继续使用 session / cookie

### 扩展优先
- 数据库 provider 可替换
- 主题机制可扩展
- 关键事件通过事件总线（发布订阅）暴露

---

## 官方支持基线

- 数据库：SQLite、PostgreSQL
- 主题：light、dark
- 其余数据库 / 主题能力留给后续扩展

---

## 验收口径

满足以下条件即可视为当前阶段达标：

1. 可写：Markdown 文章创建、编辑、发布
2. 可看：首页、详情、分类、标签访问稳定
3. 可管：分类标签与作者资料可维护
4. 可插图：图片上传并在文章中可见
5. 可扩展：数据库与主题具备明确扩展入口
